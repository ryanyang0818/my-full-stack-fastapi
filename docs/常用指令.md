# 常用指令

## Docker 相關指令

| 分類 | 指令 | 用途 |
|---|---|---|
| 查看 Compose 專案 | `docker compose ls --all` | 看目前有哪些 Docker Compose project |
| 查看服務狀態 | `docker compose ps -a` | 看 `backend-1`、`db-1` 等服務是否啟動 |
| 啟動服務 | `docker compose up -d db backend` | 啟動指定服務 |
| 重建 Backend | `docker compose up -d --force-recreate backend` | `.env` 改完後，讓 backend 重新吃設定 |
| 看 Backend log | `docker logs --tail 120 full-stack-fastapi-template-backend-1` | 查看 backend 最近錯誤 |
| 看 DB log | `docker logs --tail 80 full-stack-fastapi-template-db-1` | 查看 PostgreSQL 狀態 |
| 查看所有 container | `docker container ls -a` | 看包含已停止的 container |
| 刪除錯誤 container | `docker rm <container-name>` | 刪掉誤啟動的裸 container |
| 進入 backend container | `docker compose exec backend bash` | 進 backend shell |
| 健康檢查 | `curl http://localhost:8000/api/v1/utils/health-check/` | 確認 backend API 是否活著 |

## Docker 注意事項

| 情境 | 建議 |
|---|---|
| 改 `.env` | 用 `docker compose up -d --force-recreate backend` |
| 不要從 Images 直接按 backend image 的 Play | 會變成裸 container，沒有 compose 注入的 `.env` |
| 不要亂刪 `db-1` / volume | 會影響資料庫資料 |
| `prestart-1` exited | 正常，這通常是初始化任務跑完退出 |

## Frontend / Bun 指令

| 分類 | 指令 | 用途 |
|---|---|---|
| 安裝依賴 | `bun install` | 依照 `bun.lock` 安裝套件 |
| 啟動前端 | `bun run dev` | 啟動 Vite dev server |
| 指定 port 啟動 | `bun run --filter frontend dev -- --host 127.0.0.1 --port 5174` | 在指定 port 測 worktree 版本 |
| 前端 lint | `bunx biome check frontend/src --no-errors-on-unmatched --files-ignore-unknown=true` | 檢查前端格式與 lint |
| 單檔檢查 | `bunx biome check frontend/src/routes/_layout/index.tsx --no-errors-on-unmatched --files-ignore-unknown=true` | 只檢查指定檔案 |
| 前端測試 | `bun run test` | 執行 Playwright 測試 |
| 可見瀏覽器測試 | `bun run test -- --headed` | 執行 Playwright headed 測試，會開啟瀏覽器視窗 |
| 開 CMD 跑 headed 測試 | `Start-Process -FilePath 'cmd.exe' -ArgumentList '/k', 'bun run test -- --headed' -WorkingDirectory 'R:\001.工作日誌\05-w23\my-full-stack-fastapi\frontend'` | 從 PowerShell 開啟可見 CMD 視窗並在 `frontend` 目錄執行測試 |

## Backend / Python 指令

| 分類 | 指令 | 用途 |
|---|---|---|
| 進 backend container | `docker compose exec backend bash` | 進入後端環境 |
| 後端測試 | `docker compose exec backend pytest` | 在 container 內跑 pytest |
| 查看 migration | `docker compose exec backend alembic history` | 看 Alembic migration 歷史 |
| 套用 migration | `docker compose exec backend alembic upgrade head` | 更新資料庫 schema |

## Git 指令

| 分類 | 指令 | 用途 |
|---|---|---|
| 查看狀態 | `git status --short --branch` | 看目前分支與未提交檔案 |
| 查看差異 | `git diff` | 看尚未提交的修改 |
| 查看單檔差異 | `git diff -- path/to/file` | 只看某個檔案 |
| 查看最近 commit | `git log --oneline --decorate -8` | 看最近 commit |
| 查看 commit 內容 | `git show --stat --oneline HEAD` | 看最新 commit 改了什麼 |
| 還原單檔 | `git restore path/to/file` | 還原指定檔案，使用前要確認不是使用者改的 |

## 搜尋 / 檔案檢查

| 分類 | 指令 | 用途 |
|---|---|---|
| 搜尋文字 | `rg "keyword"` | 快速搜尋專案文字 |
| 搜尋檔案 | `rg --files` | 列出專案檔案 |
| PowerShell 讀檔 | `Get-Content -LiteralPath "path" -Encoding UTF8` | 讀 UTF-8 檔案 |
| 查看資料夾 | `Get-ChildItem -Force` | 列出檔案與隱藏檔 |
| 查看指定 port | `Get-NetTCPConnection -LocalPort 5173 -State Listen` | 查哪個 process 佔用 port |

## 收錄原則

| 類型 | 是否收錄 |
|---|---|
| 常用啟動 / 重啟 / 除錯指令 | 要 |
| 有風險但常用的指令 | 要，但加注意事項 |
| 一次性查詢指令 | 可放補充區 |
| 破壞性指令 | 要標註危險，例如 `docker rm`、`git restore` |
| 很少用的底層指令 | 不放，避免清單太雜 |
