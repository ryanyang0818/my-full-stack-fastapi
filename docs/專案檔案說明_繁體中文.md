# 專案檔案說明

這份文件說明 root 目錄下主要檔案與資料夾的用途。順序依照原始檔案清單排列，不另外自動歸類。

## Root 檔案與資料夾

| 名稱 | 類型 | 說明 | 目前是否要先懂 |
|---|---|---|---|
| `.copier` | 資料夾 | Copier template 相關檔案，包含產生專案後更新 `.env` 的腳本。只有用 Copier 產生新專案時才重要。 | 暫時可略過 |
| `.env` | 設定檔 | 專案環境變數，例如 domain、資料庫密碼、secret key、first superuser 等。Docker Compose 會讀它。 | 要知道用途 |
| `.git` | 資料夾 | Git repository 的內部資料夾，保存 commit 歷史、branches、remotes、index 等版本控制資料。通常是 hidden folder，不要手動修改。 | 知道用途即可 |
| `.gitattributes` | Git 設定 | 控制 Git 對特定檔案的處理方式，例如換行或語言偵測。 | 可略過 |
| `.github` | 資料夾 | GitHub 專用設定，包含 GitHub Actions workflows、Dependabot、labeler 等。 | 暫時可略過 |
| `.gitignore` | Git 設定 | 告訴 Git 哪些檔案不要加入版本控制，例如暫存檔、環境檔、build 輸出。 | 要知道用途 |
| `.pre-commit-config.yaml` | 工具設定 | `prek` / pre-commit hooks 的設定，會在 commit 前跑 lint、format、YAML / TOML 檢查等。 | 改 code 前再看 |
| `.vscode` | 資料夾 | VS Code 專案設定，包含建議 extensions 與 debug launch 設定。 | 用 VS Code 時再看 |
| `backend` | 資料夾 | FastAPI 後端程式碼，包含 API routes、models、database、tests、Dockerfile。 | 重要 |
| `bun.lock` | Lockfile | Frontend / workspace 的 Bun dependency lockfile，鎖定實際安裝版本。 | 知道用途即可 |
| `compose.override.yml` | Docker Compose 設定 | 本機開發用覆寫設定，例如 ports、live reload、Mailcatcher、Traefik local proxy。 | 重要 |
| `compose.traefik.yml` | Docker Compose 設定 | 正式部署時 public Traefik proxy 用的 Compose 檔案。 | 本機可略過 |
| `compose.yml` | Docker Compose 設定 | 主要 Docker Compose 設定，定義 db、backend、frontend、adminer、prestart 等 services。 | 重要 |
| `CONTRIBUTING.md` | 文件 | 貢獻專案的說明，給想對 upstream template 提 PR 的人看。 | 可略過 |
| `copier.yml` | Copier 設定 | 定義 Copier 產生新專案時會問哪些問題、預設值與 template 設定。 | 暫時可略過 |
| `deployment.md` | 文件 | 英文部署文件，說明遠端伺服器、Traefik、DNS、GitHub Actions CD 等。 | 本機可略過 |
| `deployment_繁體中文.md` | 文件 | `deployment.md` 的繁體中文翻譯版。 | 上線前再看 |
| `development.md` | 文件 | 英文本機開發文件，說明 Docker Compose、localhost URLs、logs、Mailcatcher、local dev server。 | 重要 |
| `development_繁體中文.md` | 文件 | `development.md` 的繁體中文翻譯版。 | 重要 |
| `frontend` | 資料夾 | React 前端程式碼，包含 Vite、TypeScript、Tailwind CSS、shadcn/ui、Playwright tests。 | 重要 |
| `hooks` | 資料夾 | Copier hook 腳本，目前包含 `post_gen_project.py`，用於 template 產生後的處理。 | 暫時可略過 |
| `img` | 資料夾 | README 用的圖片，例如 login、dashboard、API docs 截圖。 | 可略過 |
| `LICENSE` | 授權文件 | 專案授權條款，這個 template 使用 MIT license。 | 知道即可 |
| `package.json` | Frontend workspace 設定 | Root package 設定，定義 Bun workspaces 與常用 scripts，例如 `dev`、`lint`、`test`。 | 要知道用途 |
| `pyproject.toml` | Python 專案設定 | Root Python project 設定，和 uv workspace / backend dependency 管理有關。 | 後端開發再看 |
| `README.md` | 文件 | 英文主 README，介紹 tech stack、使用方式、Copier、部署與開發文件入口。 | 已有翻譯可先不看 |
| `README_Traditional_Chinese.md` | 文件 | `README.md` 的繁體中文翻譯版。 | 建議先看 |
| `release-notes.md` | 文件 | 專案 release notes，記錄 template 歷史版本變更。 | 可略過 |
| `scripts` | 資料夾 | 專案輔助腳本，例如測試、產生 frontend client、更新 release date。 | 需要時再看 |
| `uv.lock` | Lockfile | Python dependencies 的 uv lockfile，鎖定後端實際依賴版本。 | 知道用途即可 |
| `專案檔案說明_繁體中文.md` | 文件 | 目前這份 root 目錄檔案說明文件，用來快速理解各檔案 / 資料夾用途。 | 重要 |
