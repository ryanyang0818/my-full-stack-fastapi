Execution roadmap

用 6 週，把骨架變成可對客 demo 的痛點流程。

地基層（Auth／CI／Docker／User Management）已成熟；缺的是「對外展示層」與三支柱中的兩支—— Config-driven 與 Role Permission。本路線圖以「庫存 + 客戶銷售」兩個業務域為載體， 一次點亮 Dashboard、Data Tables、Role Permission，並驗證 config 骨架能吃下第二個業務域。

範圍：6 週壓縮版 載體：庫存 + 客戶銷售 執行：單兵 + 同時跑訪談 依據：5 份程式碼稽核 + 對抗驗證

Where the code stands

程式碼現況：地基穩，展示層空。

地基層 · 成熟可信

Auth（JWT / Argon2 / timing 防護 / 找回密碼）、CI 三條 workflow + 後端覆蓋率 90%、Docker 一鍵起、User Management 前後端皆完整。

展示層 · 基本空殼

Dashboard 佔位、ERP Demo 只有 Item 佔位、Data Tables/Forms 手刻且無查詢、RBAC 有模型無 enforcement、seed 未自動化。

六大模組真實狀態

  • User ManagementDONE
  • Data Tables共用殼但手刻、無搜尋/篩選/排序PARTIAL
  • Forms欄位散在 columns / Add / Edit 三處PARTIAL
  • Dashboard33 行靜態歡迎詞STUB
  • Role Permission有模型，但 API 無 enforcement、無管理 UISTUB
  • ERP Demo只有 Item(title/description)STUB

Target data model

兩業務域，一個外鍵把它們串起來。

庫存域證明「關聯 + 查詢」；客戶銷售域補上「錢 + 主從關聯」。關鍵在 SalesOrderItem.product_id → Product：客戶下訂單，明細直接引用庫存商品。

同域外鍵 跨域引用（訂單明細 → 庫存商品） ↗ = 外鍵

Permission story

權限故事：域級授權，真的擋得住。

複用既有 seed 的角色，不新建。授權收在域級 require_menu dependency（W6 啟用），demo 中切換角色會看到不同選單、不同可存取資源。

Scope red-lines

三條不可失守的 YAGNI 紅線。

金額不滑進會計

單一幣別 TWD、無稅率欄、subtotal = qty × unit_price 純展示、total_amount 唯讀加總、tax_id 只存不算。

狀態不變簽核

status 是自由可改純欄位，無狀態機、無轉移守衛、無鎖定。不可逆邏輯留未來。

下單不碰庫存

下單不讀 Stock.qty、不校驗、不警告；Product 下拉只取 name / sku。自動扣庫存屬複雜商業規則，排除。

另一條架構紀律：新域 CRUD 不抄 owner_id 列級隔離，授權統一收在域級 require_menu，避免兩套授權心智打架。

6-week timeline

6 週時間軸。

W1 地基 + 庫存主檔 + 權限介面早定
  • seed 串進 prestart（冪等已驗）
  • Product / Warehouse CRUD
  • no-op require_menu() + 查詢契約

Exit：全新環境 up 後選單樹自動出現、可登入

W2 庫存查詢 + 種子
  • Stock 關聯表
  • q + 單欄 sort + 真分頁
  • 庫存 demo 種子（>1 頁）

Exit：Swagger 帶參數回正確子集 + total

W3 ⚠ 最高風險週 ResourceConfig 試點（夠用就好）
  • 第一版只 text/number/select/date
  • Stock 查詢前端 toolbar
  • 重生 sdk + tsc 全綠

Exit：改一欄只動一處；列表可搜尋翻頁

W4 庫存點亮 + 客戶主檔
  • config 補 relation 型別
  • Customer 用 config 套出（證明複用）
  • Dashboard 精簡 1 張 KPI 卡

Exit：新業務域 = 動 config

W5 銷售訂單 master-detail（錢）
  • SalesOrder + 明細（守三紅線）
  • 訂單頭 + 明細 bespoke 頁
  • 明細引用庫存商品、算總額

Exit：可建含多筆明細訂單、總額正確

W6 權限可信化 + 交付收尾
  • require_menu 換真 → 跨域 403
  • 最小角色指派 UI
  • 品牌化 + README + 業務 E2E

Exit：跨域直打回 403；E2E 綠燈

6 週壓縮取捨（相對 7 週版）：砍 StockAdjustment · Dashboard KPI 精簡為 1 張卡 · 業務 E2E 收斂為 1 條 happy path + 1 條跨角色 403。 不砍 seed→prestart 與 require_menu——後者是相對 tiangolo 模板的唯一硬差異化。
