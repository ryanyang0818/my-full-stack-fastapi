# Dodo QuickSwitch Chrome 擴充套件

簽名：Claude Sonnet 4.6
建立日期：2026-06-29

## 概念

Admin 專屬的開發輔助工具：在登入頁右上角浮出一個下拉選單，列出 11 個角色帳號（標籤帶角色名稱，例如「總經理」）。選一個帳號，套件自動把 email、password 填入登入表單並送出，省去每次手動切換角色測試時重複輸入帳密的麻煩。

限定只在登入頁生效，不影響登入後的任何頁面，降低開發與安全風險。考量本機開發時 port 可能因衝突而漂移，`matches` 涵蓋 `localhost:5173` ~ `5179`（manifest 不支援 port 範圍語法，需逐一列出 7 個 port）。

## 範圍

- 純前端 Chrome Extension（Manifest V3），不修改 `backend/`、`frontend/` 任何既有檔案
- 帳密清單寫死在套件內的獨立檔案，不透過 API 取得（後端本來就不會回傳密碼）
- 僅供本機開發環境使用，不上架 Chrome 商店，以「載入未封裝項目」方式安裝

## 資料夾結構

新增於專案根目錄，獨立於 `frontend/`，避免被 bun/vite 建置流程處理：

```
extensions/dodo-quickswitch/
├── manifest.json     # Manifest V3，content_scripts.matches 限定 http://localhost:5173~5179/login*
├── content.js        # 注入下拉選單 + React controlled input 安全填值 + 自動送出
└── accounts.js       # 11 個帳號的 label/email/password 清單
```

## 實作要點

1. **manifest.json**
   - `manifest_version: 3`
   - `content_scripts[0].matches` 逐一列出 5173~5179 七個 port：
     ```json
     "matches": [
       "http://localhost:5173/login*",
       "http://localhost:5174/login*",
       "http://localhost:5175/login*",
       "http://localhost:5176/login*",
       "http://localhost:5177/login*",
       "http://localhost:5178/login*",
       "http://localhost:5179/login*"
     ]
     ```
   - `run_at: "document_idle"`（等 React 把表單渲染出來再注入）

2. **content.js**
   - 確認頁面上存在 `[data-testid="email-input"]`、`[data-testid="password-input"]`，找不到就不執行（避免頁面結構改了套件報錯）
   - 在 `document.body` 注入一個固定定位（`position: fixed; top; right`）的浮動面板，內含 `<select>`，選項文字 = 帳號標籤
   - 填值不能直接 `input.value = x`（React controlled input 不會偵測到），要用：
     ```js
     const nativeInputValueSetter = Object.getOwnPropertyDescriptor(
       window.HTMLInputElement.prototype, "value"
     ).set
     nativeInputValueSetter.call(input, value)
     input.dispatchEvent(new Event("input", { bubbles: true }))
     ```
   - 填完 email + password 後，找 `form` 內的 `button[type="submit"]` 並 `.click()`

3. **accounts.js**
   - 純資料陣列：`[{ label: "總經理", email: "...", password: "changeOK" }, ...]`
   - `content.js` 用 `<script>` 標籤或 ES module 方式載入這份資料（Manifest V3 content script 可用 `"js": ["accounts.js", "content.js"]` 依序載入，`accounts.js` 把陣列指定到 `window.__DODO_ACCOUNTS__`）

## 不做的事

- 不做帳密加密儲存（純開發工具，假設本機環境是安全的）
- 不做多環境網址切換 UI（網址寫死在 `manifest.json`，要換環境就改設定檔重新載入套件）
- 不上架 Chrome Web Store
