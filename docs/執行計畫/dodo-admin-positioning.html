Code-first internal system skeleton

Dodo Admin 是一套可落地、可擴充的內部營運後台骨架。

它不是完整 ERP、MES 或 No Code 平台，而是以工程師交付為核心的產品骨架。 透過 Dashboard、權限、資料表、表單與情境 Demo，協助小型團隊把 Excel、紙本、Line 與人工追蹤整理成可維護的內部系統。

Product definition

產品本體：不是大系統，而是可被落地的後台骨架。

Dodo Admin 的價值不是取代所有 ERP、MES 或 CMS，而是讓工程師用穩定底座快速建立某一類內部營運系統。

我幫小型製造業與傳產團隊，把最痛的內部流程，從 Excel、紙本、Line 和人工追蹤，落地成可維護、可擴充、可 self-host 的內部營運系統。
01

Code-first

保留工程師直接修改程式、資料結構與流程的能力，不被平台規則綁死。

02

Config-driven

透過設定檔定義模組、表單、資料表、導覽與權限，降低每次客製的重複成本。

03

Service-led

把訪談、流程盤點、落地計畫與系統交付包成服務，而不是只賣一套軟體。

Beachhead market

第一市場：小型製造業 / 傳產內部營運工具。

目標不是一開始服務所有產業，而是先鎖定痛點明確、預算有限、願意從單一流程開始改善的小型團隊。

目標客群

10-50 人的小型製造 / 傳產團隊，通常由老闆、主管、行政或現場管理者主導改善。

資訊分散 狀態不明 責任難追 人工彙整 流程靠經驗

第一階段痛點

  • Excel 檔案版本混亂，資料無法被穩定查詢與追蹤。
  • 紙本與 Line 訊息分散，重要狀態常常靠人記、靠人問。
  • 老闆或主管想知道進度，但缺乏一個可信的營運總覽。
  • 現有大系統太重、太貴，導入前置溝通成本過高。

Differentiation

差異化不只在產品，而是在落地工程師的服務能力。

Dodo Admin 不主張功能比大系統更多，而是用更貼近現場、更快溝通、更低導入門檻的方式解決第一個痛點流程。

Step 01

訪談

與老闆、主管或現場人員確認真正卡住的流程。

Step 02

盤點

整理資料來源、表單、權限、責任人與目前 workaround。

Step 03

計畫

提出可施行的 30 / 60 / 90 天落地方案。

Step 04

交付

用 Dodo Admin 骨架快速做出可用的第一版系統。

Step 05

擴充

把成功流程沉澱成模組，再延伸下一個痛點。

Product boundary

第一階段只處理看得到、查得到、追得到的流程。

產品邊界不是降低野心，而是避免在第一版被完整 ERP、MES 或客製顧問案拖垮。

第一階段要做

  • Dashboard：營運狀態總覽與 Demo 呈現。
  • User Management：使用者與帳號管理。
  • Role Permission：角色、權限與功能可見性。
  • Data Tables：資料查詢、維護、匯出與狀態檢視。
  • Forms：資料輸入、表單流程與紀錄起點。

第一階段不做

  • 不承諾完整 ERP / MES 範圍。
  • 不做複雜生產排程與即時設備連線。
  • 不做完整財務會計與法規報表。
  • 不把產品定位成完全自動生成的 No Code 平台。
  • 不一次滿足所有部門與所有流程。

Implementation rhythm

用 30 / 60 / 90 天，把討論變成可執行計畫。

對客戶來說，這是一個降低風險的導入節奏；對產品來說，這也是把客製需求沉澱回骨架的節奏。

前 30 天：流程盤點

訪談 5-10 位相關人員，選出一個最痛、最可落地、最能展示價值的流程。

第 31-60 天：第一版落地

用 Dodo Admin 建立資料表、表單、權限與狀態總覽，交付可操作 Demo。

第 61-90 天：擴充與驗證

根據實際使用情況補強流程，整理下一個可擴充模組與後續維護方式。

一句話收斂

Dodo Admin 是一套由落地工程師驅動的內部營運系統骨架，專注協助小型團隊從最痛的流程開始，把分散在 Excel、紙本、Line 與人工追蹤中的資訊，逐步整理成可維護、可擴充、可自有部署的後台系統。
