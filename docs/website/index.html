企業後台管理系統的基礎骨架

非完整 ERP
01 · 產品意圖

Dodo Admin 是企業後台管理系統的基礎骨架。

Dodo Admin 是一套給企業內部管理系統使用的 full-stack admin foundation。它的目的是先建立一個穩定、清楚、可擴充的後台系統骨架,讓後續功能可以用一致的方式被加入。

核心意圖

  • ✓提供登入、權限、選單、版面、CRUD、資料表等後台基礎能力
  • ✓讓工程師可以快速建立新的管理模組
  • ✓讓管理者可以清楚控制使用者能看到與操作的功能
  • ✓讓 AI agent 能透過文件、規則與一致架構穩定協作
邊界

Dodo Admin 不是完整 ERP。

它目前不追求複雜的會計、庫存稽核、製造流程或大型企業流程編排,而是優先把 admin 系統的共同基礎打穩。

02 · 功能規格

六項基礎能力

這些能力是未來所有管理模組都會共用的底層骨架。

  • 01

    使用者登入

    只有授權使用者可以進入後台,保護企業內部資料不被未授權存取。

  • 02

    角色與權限

    依使用者角色決定能看到哪些功能;管理者看見系統設定,一般使用者只看見自己工作相關的功能。

  • 03

    功能選單

    統一的功能選單,依權限顯示,使用者只會看到自己可以使用的功能。

  • 04

    後台版面

    一致的操作版面(頂部資訊列、左側選單、主要工作區、底部狀態列),降低每個新功能的學習成本。

  • 05

    資料新增、查看、修改、刪除

    基本資料管理能力,透過資料模型與固定操作流程管理,採參數化查詢,避免資料存取邏輯散落各處。

  • 06

    資料表操作

    提供查找、瀏覽、篩選、排序與操作資料的主要入口;資料來源與存取權限仍由後端 API 控制。

功能層級

  • 基礎平台功能:登入、權限、選單、版面、資料管理、資料表
  • 業務模組功能:商品、客戶、訂單、庫存、報表等可擴充模組

資料安全原則

  • 前端只負責操作體驗,實際資料讀寫由後端控管
  • 採用資料模型管理欄位、格式與驗證規則
  • 需要 SQL 時採參數化查詢,不直接拼接使用者輸入
  • 選單顯示是使用體驗,資料操作仍需後端權限檢查
03 · 使用流程

後台系統的共用操作流程

這些流程是所有後台管理功能都會共用的操作方式,適用於任何產業與業務模組。

  1. 登入 —— 使用者輸入帳號密碼
    系統驗證身份,登入後進入後台
  2. 找功能 —— 使用者從左側選單選擇功能
    系統開啟對應頁面或工作頁籤
  3. 看資料 —— 使用者進入資料列表
    系統以資料表呈現可查看的資料
  4. 操作資料 —— 使用者新增、編輯或刪除資料
    系統依權限與資料規則處理操作
  5. 切換工作 —— 使用者在不同頁籤間切換
    系統保留工作區狀態,方便多工
  6. 快捷鍵操作 —— 使用者透過鍵盤快速執行常用動作
    系統提供更有效率的操作方式(鍵位尚未定案,先定義為共用能力)
  7. 權限限制 —— 使用者沒有某功能權限
    系統不顯示功能入口,後端仍需控管存取
  8. 離開系統 —— 使用者登出
    系統清除登入狀態,回到登入頁

目前不包含的流程

不列入項目原因
採購流程目前尚未定義實際商業模組
訂單流程避免讓人誤以為產品已包含訂單系統
庫存流程避免將核心範圍擴大成完整 ERP
完整審批流目前先聚焦後台共用操作方式
04 · 權限說明

身份確認,不等於功能權限

登入負責確認身份,權限負責決定功能範圍。這兩件事必須分開理解,不能把「已登入」誤認成「所有功能都可以使用」。

權限設計重點
項目說明
身份確認使用者必須先登入,系統才會承認他的身份
功能權限不同角色會看到不同功能入口
選單範圍選單代表使用者可進入的功能範圍
個人例外特定使用者可以有額外開放或排除
後端檢查真正的資料操作仍由後端確認權限

角色與功能權限

Dodo Admin 採用角色導向的權限設計:先定義角色,再決定這個角色可以使用哪些功能。權限不用逐一設定在每個人身上,多數使用者可透過角色快速管理,少數特殊情況再用個人例外處理。

角色可使用功能
管理者使用者管理、選單管理、系統設定
一般使用者與自己工作相關的功能
特定職務依職務開放對應功能

個人例外(少數情況,不取代角色設計)

情境說明
額外開放某位使用者需要臨時看到特定功能
特別排除某位使用者雖屬於角色,但不應看到某功能
職務過渡使用者正在交接職務,需要短期調整功能

安全原則

身份與權限分離

登入只代表身份成立,不代表擁有所有權限

後端是權限依據

前端顯示結果不能取代後端檢查

角色優先

大多數權限應透過角色管理

例外克制使用

個人例外只處理少數特殊情況

拒絕優先

若同時有開放與排除,應以排除為準

05 · 常見問題

常見問題

Dodo Admin 是 ERP 嗎?

不是。Dodo Admin 是企業後台管理系統的基礎平台,重點是提供登入、權限、選單、版面、資料管理與資料表等共用能力。它可以支撐未來的業務模組,但本身不等於完整 ERP。

Dodo Admin 目前的核心範圍是什麼?

Dodo Admin 的核心範圍,是建立所有後台管理功能都會用到的基礎能力:登入、權限、選單、後台版面、資料管理、資料表、共用操作流程。

登入之後,所有功能都能直接使用嗎?

登入只代表系統確認使用者身份。使用者能看到哪些功能,還要依照角色、選單權限與個人例外決定。

前端隱藏選單,就代表安全了嗎?

前端隱藏選單是一種使用體驗控制,真正的資料操作仍需要由後端確認權限。

可以加入採購、訂單、庫存這類功能嗎?

可以。這些可以作為可擴充的業務模組加入,建立在 Dodo Admin 已經打穩的基礎能力之上。

為什麼文件裡會提到 AI agent?

因為 Dodo Admin 的目標之一,是讓工程師與 AI agent 都能穩定理解並接手專案。清楚的文件、固定的資料夾結構與一致的開發規則,能降低交接成本,也能讓 AI 協作更穩定。
