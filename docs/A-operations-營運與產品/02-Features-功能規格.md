# Dodo Admin 功能規格說明

Dodo Admin 的核心範圍，
是建立一套企業內部管理系統的基礎能力。

這些能力不是單一功能，
而是未來所有管理模組都會共用的底層骨架。

## 基礎功能範圍

### 1. 使用者登入

系統提供登入機制，只有授權使用者可以進入後台。

這能保護企業內部資料，避免未授權人員直接存取管理功能。

### 2. 角色與權限

系統可以依照使用者角色，決定他能看到哪些功能。

例如管理者可以看到使用者管理與系統設定，
一般使用者只看到與自己工作相關的功能。

### 3. 功能選單

系統提供統一的功能選單，讓使用者從左側選單進入不同管理功能。

選單會依照權限顯示，使用者只會看到自己可以使用的功能。

### 4. 後台版面

系統提供一致的後台操作版面，
包含頂部資訊列、左側選單、主要工作區與底部狀態列。

這讓每個新功能都能用相同方式加入，降低使用者學習成本。

### 5. 資料新增、查看、修改、刪除

系統支援基本資料管理能力，也就是新增、查看、修改與刪除資料。

這是多數企業內部管理流程的基礎，
例如客戶資料、商品資料、訂單資料、設定資料等。

在後端實作上，
一般資料操作會透過資料模型與固定操作流程管理，
避免讓資料存取邏輯散落在各個地方。

系統重視資料操作安全。

一般資料查詢會透過安全的查詢工具處理；
需要使用 SQL 查詢的地方，
也會採用參數化查詢，
避免將使用者輸入直接拼接成 SQL 指令。

### 6. 資料表操作

系統提供資料表介面，用來呈現大量資料。

資料表可作為查找、瀏覽、篩選、排序
與操作資料的主要入口。

資料表背後的資料來源由後端 API 控制。

前端表格只負責呈現與操作入口，
不直接決定資料存取權限。

這能確保使用者看到的資料與能執行的操作，
仍由後端的登入、權限與資料查詢規則控管。

## 功能層級

| 層級 | 說明 |
|---|---|
| 基礎平台功能 | 登入、權限、選單、版面、資料管理、資料表 |
| 業務模組功能 | 商品、客戶、訂單、庫存、報表等可擴充模組 |

## 資料安全原則

| 原則 | 說明 |
|---|---|
| 後端控管資料存取 | 前端畫面只負責操作體驗，實際資料讀寫由後端控管 |
| 使用資料模型管理資料 | 讓資料欄位、格式與驗證規則有一致依據 |
| 採用參數化查詢 | 需要使用 SQL 時，不直接拼接使用者輸入 |
| 權限與選單分離 | 選單顯示是使用體驗，資料操作仍需後端權限檢查 |

## 核心範圍

Dodo Admin 的核心範圍不是完成所有業務模組，
而是建立「所有業務模組都會用到的管理系統基礎能力」。
