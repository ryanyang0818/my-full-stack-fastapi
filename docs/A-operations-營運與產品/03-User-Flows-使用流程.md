# Dodo Admin 使用流程說明

Dodo Admin 目前描述的是後台系統的共用使用流程。

這些流程不是特定產業或特定業務模組的流程，
而是所有後台管理功能都會共用的操作方式。

## 核心使用流程

| 流程 | 使用者動作 | 系統回應 |
|---|---|---|
| 登入 | 使用者輸入帳號密碼 | 系統驗證身份，登入後進入後台 |
| 找功能 | 使用者從左側選單選擇功能 | 系統開啟對應頁面或工作頁籤 |
| 看資料 | 使用者進入資料列表 | 系統以資料表呈現可查看的資料 |
| 操作資料 | 使用者新增、編輯或刪除資料 | 系統依權限與資料規則處理操作 |
| 切換工作 | 使用者在不同頁籤間切換 | 系統保留工作區狀態，方便多工 |
| 快捷鍵操作 | 使用者透過鍵盤快速執行常用動作 | 系統提供更有效率的操作方式 |
| 權限限制 | 使用者沒有某功能權限 | 系統不顯示功能入口，後端仍需控管存取 |
| 離開系統 | 使用者登出 | 系統清除登入狀態，回到登入頁 |

## 快捷鍵操作

Dodo Admin 會保留快捷鍵系統，
讓熟悉系統的使用者能更快完成常用操作。

快捷鍵不是取代畫面操作，
而是提供進階使用者更有效率的操作路徑。

目前這份文件先把快捷鍵定義為共用能力，
不列出尚未穩定的具體鍵位。

## 不包含的流程

| 不列入項目 | 原因 |
|---|---|
| 採購流程 | 目前尚未定義實際商業模組 |
| 訂單流程 | 避免讓人誤以為產品已包含訂單系統 |
| 庫存流程 | 避免將核心範圍擴大成完整 ERP |
| 完整審批流 | 目前先聚焦後台共用操作方式 |
