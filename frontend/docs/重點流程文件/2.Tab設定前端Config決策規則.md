# Tab 設定前端 Config 決策規則

> 本文件整理 Tab 行為設定的邊界。原則是:預設單開,只有明確需要多開的功能才額外設定。

## 一、核心規則

| 類型 | 放置位置 | 判斷 |
|---|---|---|
| 前端行為契約 | 前端 config | 跟 route、component、UI 互動強綁定 |
| 使用者本機狀態 | localStorage | 只影響目前裝置,不需要跨裝置同步 |
| 權限與資料可見性 | 後端 / DB | 前端不能作為安全邊界 |
| 營運即時可調設定 | DB | 需要後台不用發版就能調整 |
| 使用者或租戶個別化 | DB 或 profile API | 每個人或每個租戶可能不同 |

## 二、決策流程

```md
這個設定會影響權限或資料安全嗎?
→ 是:後端 / DB
→ 否

這個設定需要後台即時調整嗎?
→ 是:DB
→ 否

這個設定需要依使用者或租戶不同嗎?
→ 是:DB 或 profile API
→ 否

這個設定是否跟 route、component、UI 行為強綁定?
→ 是:前端 config
→ 否:再判斷是否屬於 domain config
```

## 三、Tab 行為設定

Tab 開啟模式屬於前端行為契約,目前不需要後台即時調整,也不需要使用者個別設定,因此放在前端 config。

| 設定 | 建議 | 理由 |
|---|---|---|
| 預設開啟模式 | single | 避免相同功能重複開太多頁籤 |
| 需要多開的功能 | 額外標記 multi | 例如明細、編輯、比對類頁面 |
| Tab 是否可關閉 | 前端 config | UI 行為 |
| Tab 是否 keep alive | 前端 config | 前端掛載策略 |
| 可看見哪些 menu | 後端 / DB | 權限資料 |

## 四、建議資料形狀

```ts
type TabOpenMode = 'single' | 'multi'

type RouteTabConfig = {
  title: string
  openMode?: TabOpenMode
  keepAlive?: boolean
  closable?: boolean
}
```

`openMode` 預設視為 `single`,只有需要多開時才寫 `openMode: 'multi'`。

## 五、開啟流程

```md
點選選單或觸發開啟頁面
→ 找到 route tab config
→ 沒有設定 openMode 時視為 single
→ single:已有相同 route tab 時切換到既有 tab
→ multi:建立新的 tabId 並新增 tab
```
