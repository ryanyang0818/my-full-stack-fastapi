# Tab Header 版面與上限切回修正計畫

**日期：** 2026-06-20  
**作者：** Codex GPT-5  
**狀態：** 計畫中，尚未實作

## 一、目標

針對 `http://localhost:5173/` 的 Tab Header 區域修正以下問題：

| 項目 | 目標 |
|---|---|
| Tab Header 位置 | 靠左、靠上，貼齊 main content 起點 |
| 底線寬度 | 灰色底線從 main content 最左側延伸到最右側 |
| X hover | 小叉叉 hover 時呈現正方形黑色底色，不吃滿整個 Tab 高度 |
| 上限切回 | 頁籤已達 20 個時，從 Router 頁面仍可切回 Tab 工作區 |
| ScrollArea 寬度 | Tab Header 右側吃滿 main content 寬度 |
| 水平捲動 | Tab 太多時不撐爆整頁 X 軸 scroll，水平捲動限制在 Tab Header 內部 |

## 二、現況判斷

| 問題 | 目前可能原因 |
|---|---|
| Tab Header 不夠靠左上 | `main` 與 Tab Header 內層都有 padding / max-width 影響 |
| 灰線不滿版 | `ScrollArea` 目前只依內容寬度或受父層寬度限制 |
| X hover 高度過高 | close button 跟 Tab chip 一起 `items-stretch`，hover 區域被拉高 |
| 滿 20 後無法回 Tab 頁 | `useOpenTab()` 遇到 `openTab()` 失敗後直接 `return`，沒有執行 `navigate({ to: '/' })` |
| 右側不滿版 | Tab Header container / `ScrollArea` 缺少穩定的 `w-full min-w-0` 約束 |
| Tab 太多時撐爆整頁 | 外層 flex / content wrapper 可能缺少 `min-w-0`、`overflow-hidden`，導致水平 overflow 往整頁擴散 |

## 三、預計影響檔案

| 檔案 | 預計變更 |
|---|---|
| `src/components/TabHeader/TabHeader.tsx` | 調整 `ScrollArea`、Tab row、close button hover 樣式 |
| `src/components/TabHeader/TabHeaderController.tsx` | 調整 Tab Header 外層 wrapper，讓 Header 寬度吃滿並貼齊上方，並限制水平 overflow |
| `src/hooks/useOpenTab.ts` | 修正達 20 個頁籤時仍可切回 `/` Tab 工作區 |
| `src/routes/_layout.tsx` | 暫不改；只有在 `max-w-7xl` 明確卡住滿版時才評估最小調整 |
