# 畫面區塊介紹

> 說明登入後共用版型 `src/routes/_layout.tsx` 的畫面區塊劃分，
> 以及其 `<Outlet />` 內首頁（頁籤系統）的區塊劃分。

## 畫面結構示意圖

```
┌──────────────────────────────────────────────────────────────────────┐
│ AppMenuBar                                                            │ 2rem  ─┐ 固定頂欄
├────────────────────────────────────────────────────────────────────--┤        │ <header>
│ UserHeader                                                            │ 3rem  ─┘ (fixed top)
├───────────────┬────────────────────────────────────────────────────────┤
│               │ TabHeader                                              │
│               │ [Dashboard] [Items ×] [Items-2 ×] [+]                  │  ← 頁籤標題列
│               ├────────────────────────────────────────────────────────┤
│  AppSidebar   │                                                        │
│               │   TabPageHost  (依 activeId 渲染單一頁籤內容)            │
│  - 導覽選單    │   ┌──────────────────────────────────────────────┐    │
│  - 可收合      │   │                                                │    │
│               │   │   DashboardView  或  ItemsView ...              │    │
│               │   │   (Outlet 實際載入內容)                          │    │
│               │   │                                                │    │
│               │   └──────────────────────────────────────────────┘    │
│               │                                                        │
├───────────────┴────────────────────────────────────────────────────────┤
│ AppFooter（依 onLayoutVisibilityChange 決定是否佔位）                    │ 1.75rem
└──────────────────────────────────────────────────────────────────────┘
```

## 一、共用版型（`_layout.tsx`）

| 區塊 | 元件 / 位置 | 說明 |
|---|---|---|
| 頂欄（固定） | `<header>`（[_layout.tsx:100-103](../src/routes/_layout.tsx#L100)） | `fixed inset-x-0 top-0`，內含兩層 |
| ├ App 選單列 | `AppMenuBar`（`src/components/Header/AppMenuBar.tsx`） | 高度變數 `--app-menu-bar-height`（2rem），由 `appMenuVisible` 控制顯隱 |
| └ 使用者標頭 | `UserHeader`（`src/components/Header/UserHeader.tsx`） | 高度變數 `--user-header-height`（3rem），由 `userHeaderVisible` 控制顯隱 |
| 側邊欄控制橋接 | `SidebarControllerBridge`（`src/components/Sidebar/SidebarControllerBridge.tsx`） | 不渲染可見 UI，把 sidebar 狀態接到外部 controller（console 介面） |
| 側邊欄 | `AppSidebar`（`src/components/Sidebar/AppSidebar.tsx`） | 主導覽選單 |
| 中央內容區 | `MainContentController`（[_layout.tsx:106-112](../src/routes/_layout.tsx#L106)） | `<main data-dodo-section="mainContent">`，內部 `<div class="max-w-7xl">` 包裹 `<Outlet />`；同時註冊到 `dodo-controller` 供外部查詢 DOM 狀態 |
| 頁腳（依需求顯示） | `AppFooter`（`src/components/Footer/AppFooter.tsx`） | 高度變數 `--app-footer-height`（1.75rem），透過 `onLayoutVisibilityChange` 回報是否佔位 |

### 補充機制

- `HeaderVisibilityProvider`（[_layout.tsx:121](../src/routes/_layout.tsx#L121)）包住整個 `AuthenticatedLayout`，控制頂欄兩段的顯隱狀態。
- 版面高度透過 CSS variables 動態計算 padding，讓固定頂欄 / 頁腳不擋住內容。
- `<Outlet />` 才是真正載入首頁（`_layout/index.tsx`，即下方頁籤系統）的位置。

## 二、首頁內容（`_layout/index.tsx`，頁籤系統）

首頁由 `TabHeaderController` 搭建「頁籤工作區」架構，頁籤內容脫離 Router，由 Zustand `tabStore` 驅動狀態管理。

| 區塊 | 元件 / 位置 | 說明 |
|---|---|---|
| 頁籤標題列 | `TabHeader`（`src/components/TabHeader/TabHeader.tsx`） | 顯示所有開啟的頁籤，支援拖拉重排、關閉右側 / 全部關閉（dnd-kit） |
| 頁籤內容渲染區 | `TabPageHost`（`src/components/TabHeader/TabPageHost.tsx:5-20`） | 只渲染當前 `activeId` 對應的頁籤內容，切換時卸載舊元件（無 keep-alive） |
| Dashboard 頁籤（固定首頁籤） | `DashboardView`（`src/components/Dashboard/DashboardView.tsx:10-31`） | 公告區（Megaphone 圖示）+ 歡迎訊息 + 使用者全名 / Email 問候 |
| Items 頁籤（可重複開啟） | `ItemsView`（`src/components/Items/ItemsView.tsx:48-62`） | 標題列 + 「Add Item」按鈕 + Suspense 包裹的 Items 資料表（DataTable）+ 空狀態提示 |

### 頁籤系統特性

- Dashboard 頁籤固定、不可關閉、永遠在最左。
- 頁籤狀態存 localStorage（`dodo.tabs.v2`），重整後還原順序與 active 狀態。
- 上限 20 個頁籤。
- 所有頁籤種類集中註冊於 `tabRegistry.ts`。
