# 常用指令

## Docker 相關指令

| 分類 | 指令 | 用途 |
|---|---|---|
| 查看 Compose 專案 | `docker compose ls --all` | 看目前有哪些 Docker Compose project |
| 查看服務狀態 | `docker compose ps -a` | 看 `backend-1`、`db-1` 等服務是否啟動 |
| 啟動服務 | `docker compose up -d db backend` | 啟動指定服務 |
| 重建 Backend | `docker compose up -d --force-recreate backend` | `.env` 改完後，讓 backend 重新吃設定 |
| 看 Backend log | `docker logs --tail 120 full-stack-fastapi-template-backend-1` | 查看 backend 最近錯誤 |
| 看 DB log | `docker logs --tail 80 full-stack-fastapi-template-db-1` | 查看 PostgreSQL 狀態 |
| 查看所有 container | `docker container ls -a` | 看包含已停止的 container |
| 刪除錯誤 container | `docker rm <container-name>` | 刪掉誤啟動的裸 container |
| 進入 backend container | `docker compose exec backend bash` | 進 backend shell |
| 健康檢查 | `curl http://localhost:8000/api/v1/utils/health-check/` | 確認 backend API 是否活著 |

## Docker 注意事項

| 情境 | 建議 |
|---|---|
| 改 `.env` | 用 `docker compose up -d --force-recreate backend` |
| 不要從 Images 直接按 backend image 的 Play | 會變成裸 container，沒有 compose 注入的 `.env` |
| 不要亂刪 `db-1` / volume | 會影響資料庫資料 |
| `prestart-1` exited | 正常，這通常是初始化任務跑完退出 |

## Git 指令

| 分類 | 指令 | 用途 |
|---|---|---|
| 查看狀態 | `git status --short --branch` | 看目前分支與未提交檔案 |
| 查看差異 | `git diff` | 看尚未提交的修改 |
| 查看單檔差異 | `git diff -- path/to/file` | 只看某個檔案 |
| 查看最近 commit | `git log --oneline --decorate -8` | 看最近 commit |
| 查看 commit 內容 | `git show --stat --oneline HEAD` | 看最新 commit 改了什麼 |
| 還原單檔 | `git restore path/to/file` | 還原指定檔案，使用前要確認不是使用者改的 |
| 列出 worktree | `git worktree list` | 看目前有哪些工作區與對應分支 |
| 移除 worktree | `git worktree remove "<worktree-path>"` | 刪除工作區資料夾；有未提交變更會被擋下 |
| 強制移除 worktree | `git worktree remove --force "<worktree-path>"` | 連同未提交變更一起刪掉工作區，危險 |
| 刪除分支 | `git branch -D <branch-name>` | 強制刪除分支，**未合併的 commit 會永久消失**，危險 |

### 連同 worktree 與 branch 一起刪除（須在主 repo 執行，不可在 worktree 內）

```bash
# 1. 移除 worktree（刪掉工作區資料夾）
git worktree remove "R:/001.工作日誌/05-w23/my-full-stack-fastapi/frontend/.claude/worktrees/sharp-elbakyan-6a89a5"

# 2. 刪掉對應的 branch
git branch -D claude/sharp-elbakyan-6a89a5
```

## GIT 問題排除

### 症狀：TortoiseGit 顯示 `Get refs failed: libgit2 returned: the requested type does not match the type in the ODB`

**原因**：某個 ref 指向的物件型別跟預期不符（例如該是 `commit` 卻指到 `tree`）。常見是工具（如 Codex CLI）寫入的內部暫存 ref 壞掉，並非分支或標籤本身有問題。

**排查步驟**（注意：以下指令含 `|` 管道，直接在終端機打整行即可，不要加反斜線）：

| 步驟 | 用途 |
|---|---|
| 1. 檢查物件完整性 | 列出所有物件，`dangling commit/tree/blob` 是 rebase/amend 留下的孤兒物件，正常可忽略 |
| 2. 過濾真正錯誤 | 篩出真正的 `error:` 訊息（要小心 SHA 本身含 `bad` 等字串造成的誤判） |
| 3. 列出所有 ref 與型別 | 找出哪條 ref 的型別不對 |
| 4. 快速篩出可疑 ref | 只列出型別不是 `commit`/`tag` 的 ref，一眼定位兇手 |

```bash
# 1. 檢查物件完整性
git fsck --full

# 2. 過濾真正錯誤
git fsck --full | grep -i error

# 3. 列出所有 ref 與型別
git for-each-ref --format='%(refname) %(objecttype) %(objectname)'

# 4. 快速篩出可疑 ref
git for-each-ref --format='%(refname) %(objecttype)' | grep -v ' commit$' | grep -v ' tag$'
```

**修復**：找到壞掉的 ref 後直接刪除（只刪 ref 指標，不影響底下物件）：

```bash
git update-ref -d refs/codex/turn-diffs/captures/<id>/base
```

**還原（如果刪錯了）**：`git update-ref -d` 不會刪除物件本身，只要 `git gc` 還沒清掉未引用物件（預設保留 2 週），用刪除前記下的 SHA 就能復原：

```bash
git update-ref refs/codex/turn-diffs/captures/<id>/base <原本的 SHA>
```

## 資料夾刪不掉：查進程並強制結束（cmd）

**情境**：worktree 或 `node_modules` 資料夾刪不掉，通常是還有 `node.exe` / `esbuild.exe`（Vite dev server 及其編譯子進程）在背景跑，鎖住裡面的檔案。

**第一步：查出符合條件的 PID**（依名稱 + 指令列關鍵字篩選，可依專案路徑自行替換關鍵字）：

```cmd
wmic process where "(name='node.exe' or name='esbuild.exe') and CommandLine like '%my-full-stack-fastapi%'" get ProcessId,CommandLine
```

**第二步：用查到的 PID 強制結束進程**：

```cmd
taskkill /PID <PID1> /F
taskkill /PID <PID2> /F
```

**第三步：驗證是否真的結束**：

```cmd
tasklist /FI "PID eq <PID1>"
```
顯示「沒有符合的工作」才代表確定清掉了，這時候資料夾就能正常刪除。

**注意事項**：
- `CommandLine like '%關鍵字%'` 篩的是「執行檔路徑」，不是進程實際的工作目錄（cwd）。如果該專案的 `node_modules` 剛好就裝在這個路徑底下，篩得到；但無法保證這支進程實際服務的是哪一個 worktree。
- 如果要 100% 確定是哪個進程鎖住了特定資料夾，要用 Sysinternals **Process Explorer** 的 `Find > Find Handle or DLL...`（Ctrl+F）直接搜資料夾路徑，這才是有實證的做法。
- `wmic` 在較新版本 Windows 11 可能已被移除，若指令打不開，可改用 PowerShell 的 `Get-CimInstance Win32_Process -Filter "..."` 達到相同效果。

## 搜尋 / 檔案檢查

| 分類 | 指令 | 用途 |
|---|---|---|
| 搜尋文字 | `rg "keyword"` | 快速搜尋專案文字 |
| 搜尋檔案 | `rg --files` | 列出專案檔案 |
| PowerShell 讀檔 | `Get-Content -LiteralPath "path" -Encoding UTF8` | 讀 UTF-8 檔案 |
| 查看資料夾 | `Get-ChildItem -Force` | 列出檔案與隱藏檔 |
| 查看指定 port | `Get-NetTCPConnection -LocalPort 5173 -State Listen` | 查哪個 process 佔用 port |

## 收錄原則

| 類型 | 是否收錄 |
|---|---|
| 常用啟動 / 重啟 / 除錯指令 | 要 |
| 有風險但常用的指令 | 要，但加注意事項 |
| 一次性查詢指令 | 可放補充區 |
| 破壞性指令 | 要標註危險，例如 `docker rm`、`git restore` |
| 很少用的底層指令 | 不放，避免清單太雜 |
