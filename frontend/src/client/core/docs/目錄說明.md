# src/client/core/ 說明

`@hey-api/openapi-ts` 產生的 client 都會附帶這一組「底層引擎」檔案，跟後端 schema 內容無關，是固定的樣板程式碼（除非手動切換 client 產生器設定，否則不會變）。

## 總覽

| 項目 | 類型 | 分類 | 說明 |
|---|---|---|---|
| ApiError.ts | 檔案 | 套件架構本身 | API 回應錯誤時拋出的例外類別 |
| ApiRequestOptions.ts | 檔案 | 套件架構本身 | 描述「一個請求要怎麼發」的型別 |
| ApiResult.ts | 檔案 | 套件架構本身 | 描述「一個回應長什麼樣子」的型別 |
| CancelablePromise.ts | 檔案 | 套件架構本身 | 可中途取消的 Promise |
| OpenAPI.ts | 檔案 | 套件架構本身 | 全域設定物件（BASE 網址、TOKEN 等） |
| request.ts | 檔案 | 套件架構本身 | 真正用 axios 發送 HTTP 請求的核心函式 |

---

## 重點檔案：`OpenAPI.ts` — 整個 client 的「全域設定面板」

```ts
export type OpenAPIConfig = {
	BASE: string;
	CREDENTIALS: 'include' | 'omit' | 'same-origin';
	TOKEN?: string | Resolver<string> | undefined;
	VERSION: string;
	WITH_CREDENTIALS: boolean;
	interceptors: {
		request: Interceptors<AxiosRequestConfig>;
		response: Interceptors<AxiosResponse>;
	};
	// ...
};

export const OpenAPI: OpenAPIConfig = {
	BASE: '',
	CREDENTIALS: 'include',
	TOKEN: undefined,
	VERSION: '0.1.0',
	WITH_CREDENTIALS: false,
	interceptors: { request: new Interceptors(), response: new Interceptors() },
};
```

這是一個**全域共用的單例物件**——整個 App 只有一份。[main.tsx](../../docs/目錄說明.md) 在 App 啟動時做的兩件事：

```ts
OpenAPI.BASE = import.meta.env.VITE_API_URL
OpenAPI.TOKEN = async () => localStorage.getItem("access_token") || ""
```

就是直接**修改這個物件的屬性**。之後 [sdk.gen.ts](../../docs/目錄說明.md) 裡每一個 `XxxService.xxx()` 方法呼叫 [request.ts](#重點檔案-requestts--真正送出-http-請求的地方) 時，都會把這個 `OpenAPI` 物件傳進去，所以每一支 API 都自動帶上正確的 base URL 與 Authorization token，業務元件完全不用關心。

`TOKEN` 的型別是 `string | Resolver<string>`——也就是「可以是字串，也可以是一個會回傳字串的函式（甚至是 async 函式）」。這就是為什麼 `main.tsx` 可以把它設成一個 `async () => {...}`：每次發請求前都重新去 `localStorage` 讀一次最新的 token，而不是只在啟動時讀一次。

`interceptors` 提供 `request.use(fn)` / `response.use(fn)`，可以在「送出前」或「收到回應後」插入自訂邏輯（例如統一加 header、統一記錄 log），但目前本專案沒有使用。

## 重點檔案：`request.ts` — 真正送出 HTTP 請求的地方

這個檔案（346 行）是所有 API 呼叫最終會經過的「唯一出口」。可以把它想成一條生產線，[sdk.gen.ts](../../docs/目錄說明.md) 裡每個方法呼叫 `__request(OpenAPI, options)` 時，會依序經過：

1. **組網址**（`getUrl`）：把 `OpenAPI.BASE`（例如 `http://localhost:8000`）跟 `options.url`（例如 `/api/v1/items/{id}`）接起來，並把 `{id}` 這種路徑參數替換成實際值。
2. **組查詢字串**（`getQueryString`）：把 `options.query`（例如 `{ skip: 0, limit: 100 }`）轉成 `?skip=0&limit=100`，連陣列、物件、`Date` 都會處理。
3. **組 headers**（`getHeaders`）：合併 `OpenAPI.HEADERS`、`options.headers`，並呼叫 `resolve()` 解析 `OpenAPI.TOKEN`（如果是函式就執行它拿到字串），組成 `Authorization: Bearer <token>`。
4. **組 request body**（`getRequestBody`）：依 `options.mediaType` 決定是送 JSON、表單（`application/x-www-form-urlencoded`，例如登入用的帳密）還是 `FormData`（檔案上傳）。
5. **用 axios 發送**，並用 [CancelablePromise](#cancelablepromisets--可以中途取消的-promise) 包起來，讓呼叫端可以 `.cancel()` 取消請求。
6. **檢查回應狀態碼**：如果不是 2xx，就用回應內容組成一個 [ApiError](#apierrorts--api-錯誤的標準格式) 並 `reject` 出去；成功則 `resolve` 回應的 `body`。

業務程式碼幾乎不會直接碰這個檔案，但理解它有助於除錯：例如「為什麼 API 沒帶 token？」「為什麼上傳檔案失敗？」答案通常都在這裡的某個分支。

## 其他檔案說明

### `ApiError.ts` — API 錯誤的標準格式

```ts
export class ApiError extends Error {
	public readonly url: string;
	public readonly status: number;
	public readonly statusText: string;
	public readonly body: unknown;       // 後端回傳的錯誤內容，例如 { detail: "..." }
	public readonly request: ApiRequestOptions;
}
```

當 [request.ts](#重點檔案-requestts--真正送出-http-請求的地方) 偵測到 HTTP 狀態碼不是 2xx，就會 `throw` 一個 `ApiError`。[src/utils.ts 的 handleError](../../../docs/目錄說明.md) 接到的參數就是這個型別，從 `err.body.detail` 取出後端的錯誤訊息文字。[main.tsx](../../../docs/目錄說明.md) 也是檢查 `error instanceof ApiError && [401, 403].includes(error.status)` 來判斷要不要自動登出。

### `ApiRequestOptions.ts` / `ApiResult.ts` — 請求/回應的「資料格」

`ApiRequestOptions` 描述「發一個請求需要哪些資訊」（method、url、query、body、headers...），`ApiResult` 描述「收到的回應長什麼樣」（status、statusText、body、url、ok）。這兩個型別只是「資料的形狀定義」，本身不含邏輯，是 [request.ts](#重點檔案-requestts--真正送出-http-請求的地方) 與 [ApiError.ts](#apierrorts--api-錯誤的標準格式) 之間傳遞資訊用的介面。

### `CancelablePromise.ts` — 可以中途取消的 Promise

一般 `Promise` 一旦建立就無法中途停止。`CancelablePromise<T>` 在標準 Promise 的基礎上多了 `.cancel()` 方法：

```ts
const promise = ItemsService.readItems()
// 使用者很快切換到別的頁面，這個請求結果已經不需要了
promise.cancel()
```

呼叫 `.cancel()` 後，這個 Promise 會用 `CancelError` reject，並執行透過 `onCancel(fn)` 註冊的清理函式（例如中斷底層的 axios request）。常見情境：使用者快速切換分頁，避免「舊頁面的 API 回應晚到，把新頁面的資料蓋掉」。
