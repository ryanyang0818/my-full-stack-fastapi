# src/lib/ 說明

放置與框架無關、可在任何元件中重複使用的小工具函式。目前只有一個檔案。

| 項目 | 類型 | 分類 | 說明 |
|---|---|---|---|
| utils.ts | 檔案 | 套件架構本身 | 提供 `cn()`：合併 Tailwind className 的工具函式 |

---

## `utils.ts` — `cn()`：合併 class name 的小工具

```ts
import { type ClassValue, clsx } from "clsx"
import { twMerge } from "tailwind-merge"

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs))
}
```

寫 Tailwind 元件時常常需要「依條件決定 class」，例如：

```tsx
<button className={cn(
  "rounded-md px-4 py-2",          // 基本樣式
  variant === "danger" && "bg-destructive text-white",
  variant === "primary" && "bg-primary text-white",
  className,                        // 外部傳進來想覆寫的 class
)}>
```

這裡會遇到兩個問題，`cn()` 一次解決：

1. **`clsx`**：自動過濾掉 `false` / `undefined` / `null`，把剩下的字串接起來。上面例子如果 `variant !== "danger"`，那一段就直接被忽略，不會留下 `"false"` 這種髒字串。
2. **`twMerge`**：Tailwind 的 class 是「後寫的不一定會贏」——例如 `"px-4 px-2"` 兩個都會被套用到，可能互相打架。`twMerge` 會偵測到兩個都是 `padding-x` 類，**保留最後一個、丟掉前面的**，確保外部傳入的 `className`（通常想覆寫預設樣式）真的能生效。

幾乎所有 [components/ui/](../../components/ui/) 底下的元件，最外層的 `className` 都是用 `cn(...)` 組出來的。
