# src/components/Common/ 說明

跨頁面共用的小元件：版面骨架（`AuthLayout`、`Footer`、`Logo`）、全域錯誤畫面（`ErrorComponent`、`NotFound`）、主題切換（`Appearance`）、以及所有「列表頁」共用的表格（`DataTable`）。

## 總覽

| 項目 | 類型 | 分類 | 說明 |
|---|---|---|---|
| DataTable.tsx | 檔案 | 套件架構本身 | 通用表格元件（含分頁），[items.tsx](../../../routes/_layout/docs/目錄說明.md)、[admin.tsx](../../../routes/_layout/docs/目錄說明.md) 共用 |
| Appearance.tsx | 檔案 | 套件架構本身 | 主題（亮/暗/系統）切換下拉選單，提供 2 種版本 |
| AuthLayout.tsx | 檔案 | 套件架構本身 | 登入/註冊等認證頁的共用版面（左圖右表單） |

## 重點檔案：`DataTable.tsx` — 所有列表頁共用的表格

[items.tsx](../../../routes/_layout/docs/目錄說明.md) 與 [admin.tsx](../../../routes/_layout/docs/目錄說明.md) 的表格，最終都是呼叫這一個元件：

```tsx
<DataTable columns={columns} data={items.data} />
```

它本身**不知道資料的形狀**，完全交給 `columns`（見 [Items/columns.tsx](../../Items/docs/目錄說明.md)、[Admin/columns.tsx](../../Admin/docs/目錄說明.md)）定義「每一欄要顯示什麼」。背後用的是 [TanStack Table](https://tanstack.com/table) 的 `useReactTable`：

```tsx
const table = useReactTable({
  data,
  columns,
  getCoreRowModel: getCoreRowModel(),
  getPaginationRowModel: getPaginationRowModel(),
})
```

- `getCoreRowModel`：把 `data` 陣列轉成表格能用的「row」結構。
- `getPaginationRowModel`：自動把資料切成一頁一頁——`DataTable` 不需要自己寫分頁邏輯，呼叫 `table.setPageSize(...)`、`table.nextPage()` 就能切換。

畫面分成三段：

1. **表頭與表身**：用 `flexRender(header.column.columnDef.header, ...)` / `flexRender(cell.column.columnDef.cell, ...)`——`columnDef` 裡定義的 `header`/`cell` 可以是字串，也可以是一個 React 元件（例如 [Admin/columns.tsx](../../Admin/docs/目錄說明.md) 裡用 `<Badge>` 顯示角色）。
2. **空狀態**：`table.getRowModel().rows.length === 0` 時顯示「No results found.」。
3. **分頁列**（只在 `table.getPageCount() > 1` 時顯示）：每頁筆數的 [`<Select>`](../../ui/docs/目錄說明.md)（5/10/25/50）+ 「第 X 到 Y 筆，共 Z 筆」文字 + 四個分頁按鈕（第一頁/上一頁/下一頁/最後一頁）。

**一句話總結**：`DataTable` 是「資料 + 欄位定義」→「完整表格畫面（含分頁）」的轉換器，新增一個列表頁只需要寫一份 `columns.tsx`，不用重新刻表格與分頁。

## 其他檔案說明

### `Appearance.tsx` — 主題切換下拉選單（2 個版本）

這個檔案 export 兩個元件，都是包著 [`useTheme()`](../docs/目錄說明.md) 的 [`DropdownMenu`](../../ui/docs/目錄說明.md)，選單內容都是 Light / Dark / System 三個選項，差別只在「放在哪裡」：

- **`Appearance`**：放在 [`AuthLayout`](#authlayouttsx--認證頁的共用版面) 右上角，觸發按鈕是一顆圖示按鈕，圖示會用 `dark:` class 在太陽/月亮之間做淡入淡出動畫。
- **`SidebarAppearance`**：放進 [AppSidebar](../../Sidebar/docs/目錄說明.md) 的選單裡，外觀是一般的 `SidebarMenuButton`，圖示會依目前 `theme` 動態切換成 `Sun`/`Moon`/`Monitor`（透過一個 `ICON_MAP` 物件對照）。

兩者點擊選項時都呼叫同一個 `setTheme(...)`，狀態同步機制見 [theme-provider.tsx](../docs/目錄說明.md)。

### `AuthLayout.tsx` — 認證頁的共用版面

[login.tsx](../../../routes/docs/目錄說明.md)、[signup.tsx](../../../routes/docs/目錄說明.md) 等頁面外層都包著 `<AuthLayout>`：

```tsx
<AuthLayout>
  <LoginForm />  {/* children */}
</AuthLayout>
```

桌面（`lg:` 以上）是左右兩欄：左邊是 `Logo`（鋪滿背景的大 Logo），右邊是表單；手機版左欄會隱藏（`hidden lg:flex`），只顯示表單。右上角放 `<Appearance />`，底部放 `<Footer />`。

### `ErrorComponent.tsx` / `NotFound.tsx` — 兩種「整頁出錯」畫面

兩者結構幾乎一樣（置中的大字 + 一行說明 + 一個「回首頁」按鈕），分別對應 [`__root.tsx`](../../../routes/docs/目錄說明.md) 的 `errorComponent`（程式拋出例外時顯示「Error / Oops!」）與 `notFoundComponent`（網址找不到對應路由時顯示「404 / Oops!」）。

### `Footer.tsx` — 頁尾

純展示用：左邊顯示「DoDo ERP - {目前年份}」，右邊暫時保留 GitHub / X / LinkedIn 三個社群連結圖示（用 `react-icons`）。`socialLinks` 是一個設定陣列，`.map()` 產生連結——跟 [settings.tsx](../../../routes/_layout/docs/目錄說明.md) 的 `tabsConfig` 是同一種「資料驅動畫面」寫法。

### `Logo.tsx` — 會跟著主題換色的 DoDo ERP Logo

```tsx
const { resolvedTheme } = useTheme()
const isDark = resolvedTheme === "dark"
const fullLogo = isDark ? logoLight : logo
```

依 [`useTheme()`](../docs/目錄說明.md) 的 `resolvedTheme`，在「一般版」與「亮色版」的 SVG 之間切換（暗色背景要用亮色 Logo 才看得清楚）。`variant` prop 有三種：`"full"`（完整 Logo+文字）、`"icon"`（只有圖示）、`"responsive"`（[AppSidebar](../../Sidebar/docs/目錄說明.md) 收合時自動切成 icon，用 Tailwind 的 `group-data-[collapsible=icon]:` 選擇器控制顯示/隱藏）。`asLink` 決定是否包一層 `<Link to="/">`（[AuthLayout](#authlayouttsx--認證頁的共用版面) 裡的大 Logo 不需要可點擊，傳 `asLink={false}`）。
