# src/components/Sidebar/ 說明

[_layout.tsx](../../../routes/docs/目錄說明.md) 左側主選單的「業務層」：把 [`ui/sidebar.tsx`](../../ui/docs/目錄說明.md) 提供的積木組裝成實際 Sidebar，包含固定工作區入口與後端 `GET /api/v1/menus/tree` 控制的樹狀選單。

## 總覽

| 項目 | 類型 | 分類 | 說明 |
|---|---|---|---|
| AppSidebar.tsx | 檔案 | 套件架構本身 | 整個 Sidebar 的組裝入口，負責固定入口與後端選單 |
| TreeMenu.tsx | 檔案 | 套件架構本身 | 將 `MenuTreeNodePublic[]` 遞迴渲染成 Sidebar tree |
| Main.tsx | 檔案 | 套件架構本身 | 固定主選單項目（Dashboard / Items / Admin） |
| tree-data.ts | 檔案 | 舊版相容 | 舊的靜態選單資料，目前不再由正式 Sidebar 使用 |
| User.tsx | 檔案 | 套件架構本身 | 左下角使用者頭像與選單（設定/登出） |

## 重點檔案：`AppSidebar.tsx`

`AppSidebar` 是 Sidebar 的組裝入口，主要責任是：

| 區塊 | 責任 |
|---|---|
| `Main` | 顯示固定工作區入口：Dashboard / Items / Admin |
| `MenusService.readMyMenuTree()` | 從後端取得目前登入者可見選單 |
| `SidebarSearch` | 管理前端搜尋字串 |
| `TreeMenu` | 渲染後端選單樹 |
| `SidebarAppearance` | 主題切換 |
| `User` | 使用者資訊與登出 |

固定入口與後端樹狀選單是兩層不同責任：`Main` 保留工作區核心入口；`TreeMenu` 顯示後端授權控制的 ERP 選單。`Admin` 仍由 `currentUser.is_superuser` 決定是否顯示。

## 重點檔案：`Main.tsx`

`Main` 負責渲染固定入口。

| 項目 | 行為 |
|---|---|
| Dashboard | 聚焦固定 Dashboard 頁籤 |
| Items | 開啟新的 Items 頁籤 |
| Admin | superuser 才顯示，開啟 Admin 頁籤 |

手機版點擊固定入口後會呼叫 `setOpenMobile(false)` 收起抽屜。

## 重點檔案：`TreeMenu.tsx`

`TreeMenu` 直接接收 SDK 產生的 `MenuTreeNodePublic[]`。

| 後端欄位 | 前端用途 |
|---|---|
| `id` | React key |
| `key` | 搜尋條件之一 |
| `label` | 顯示文字 |
| `path` | 轉成 `TabKey` 後開啟頁籤 |
| `icon` | Lucide React 元件名稱 |
| `sortOrder` | 前端輔助排序 |
| `children` | 遞迴渲染子選單 |

渲染規則：

| 狀況 | 行為 |
|---|---|
| 有 `children` | 顯示為可展開節點 |
| 無 `children` 且 `path` 可對應 `TabKey` | 點擊後開啟頁籤 |
| 無 `children` 且 `path` 無法對應 `TabKey` | 顯示為禁用 |
| `icon` 空值或無法解析 | 使用 `FileText` |
| API 載入中 | 顯示 `SidebarMenuSkeleton` |
| API 失敗或無資料 | 顯示空狀態，不讓 Sidebar 崩潰 |

## Path 與 Tab 的關係

`TreeMenu` 不直接信任任意後端 `path`。它會透過 [tabKeyMap.ts](../tabKeyMap.ts) 將 `path` 轉成前端支援的 `TabKey`。

| 檔案 | 責任 |
|---|---|
| tabKeyMap.ts | 維護 `path -> TabKey` 對應 |
| useOpenTab.ts | 開啟頁籤並導回首頁工作區 |
| tabRegistry.tsx | 定義每個 `TabKey` 對應的標題、icon、內容元件 |

如果後端新增了新的 menu path，但前端尚未在 `tabKeyMap.ts` 與 `tabRegistry.tsx` 補齊，該選單會顯示為禁用。

## `User.tsx`

顯示目前登入者的頭像、姓名、email；點擊展開 `DropdownMenu`，提供兩個動作：

- 「User Settings」：導向 [settings.tsx](../../../routes/_layout/docs/目錄說明.md)。
- 「Log Out」：呼叫 [`useAuth().logout()`](../../../hooks/docs/目錄說明.md)，清除 token 並導回登入頁。

手機版點擊選單項目後會呼叫 `setOpenMobile(false)` 收起抽屜，避免畫面停在「抽屜開著但內容已切換」的狀態。
