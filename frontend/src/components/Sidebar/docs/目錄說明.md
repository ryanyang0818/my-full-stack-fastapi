# src/components/Sidebar/ 說明

[_layout.tsx](../../../routes/docs/目錄說明.md) 左側主選單的「業務層」——把 [`ui/sidebar.tsx`](../../ui/docs/目錄說明.md) 提供的積木（`Sidebar`、`SidebarMenu`...）組裝成實際的選單內容、選單項目、使用者選單。

## 總覽

| 項目 | 類型 | 分類 | 說明 |
|---|---|---|---|
| AppSidebar.tsx | 檔案 | 套件架構本身 | 整個左側選單的組裝入口 |
| Main.tsx | 檔案 | 套件架構本身 | 主選單項目（Dashboard / Items / Admin） |
| User.tsx | 檔案 | 套件架構本身 | 左下角使用者頭像與選單（設定/登出） |

---

## 重點檔案：`AppSidebar.tsx` — 整個側邊欄怎麼拼出來的

```tsx
const baseItems: Item[] = [
  { icon: Home, title: "Dashboard", path: "/" },
  { icon: Briefcase, title: "Items", path: "/items" },
]

export function AppSidebar() {
  const { user: currentUser } = useAuth()

  const items = currentUser?.is_superuser
    ? [...baseItems, { icon: Users, title: "Admin", path: "/admin" }]
    : baseItems

  return (
    <Sidebar collapsible="icon">
      <SidebarHeader>...<Logo variant="responsive" />...</SidebarHeader>
      <SidebarContent>
        <Main items={items} />
      </SidebarContent>
      <SidebarFooter>
        <SidebarAppearance />
        <User user={currentUser} />
      </SidebarFooter>
    </Sidebar>
  )
}
```

可以把這個檔案當成側邊欄的「總裝配圖」，由三個部分疊起來：

1. **`SidebarHeader`**：放 [`<Logo variant="responsive">`](../../Common/docs/目錄說明.md)——側邊欄展開時顯示完整 Logo+文字，收合成 icon 時自動只顯示小圖示（Logo 元件內部用 `group-data-[collapsible=icon]:` 處理切換，這裡不用寫額外邏輯）。
2. **`SidebarContent`**：交給 `<Main items={items} />` 畫出選單項目列表。
3. **`SidebarFooter`**：[`<SidebarAppearance />`](../../Common/docs/目錄說明.md)（主題切換）+ `<User user={currentUser} />`（使用者資訊/登出）。

**權限控制就在這裡發生**：`items` 陣列是根據 [`useAuth()`](../../../hooks/docs/目錄說明.md) 拿到的 `currentUser.is_superuser` 動態決定的——只有 superuser 才會多看到一個「Admin」選單項目。這跟 [admin.tsx](../../../routes/_layout/docs/目錄說明.md) 的 `beforeLoad` 守衛是**兩道獨立的防線**：這裡只是「不顯示連結」（UI 層面），就算使用者手動輸入網址 `/admin`，真正擋下非 superuser 的是 `admin.tsx` 的 `beforeLoad`。

`collapsible="icon"`：傳給 [`ui/sidebar.tsx`](../../ui/docs/目錄說明.md) 的 `<Sidebar>`，決定收合時的行為是「縮成一條只有 icon 的窄欄」，而不是完全消失。

## 其他檔案說明

### `Main.tsx` — 把 `items` 陣列畫成可點擊的選單

```tsx
export type Item = { icon: LucideIcon; title: string; path: string }

export function Main({ items }: MainProps) {
  const { isMobile, setOpenMobile } = useSidebar()
  const currentPath = useRouterState().location.pathname

  return (
    <SidebarGroup>
      <SidebarGroupContent>
        <SidebarMenu>
          {items.map((item) => (
            <SidebarMenuItem key={item.title}>
              <SidebarMenuButton
                tooltip={item.title}
                isActive={currentPath === item.path}
                asChild
              >
                <RouterLink to={item.path} onClick={() => isMobile && setOpenMobile(false)}>
                  <item.icon />
                  <span>{item.title}</span>
                </RouterLink>
              </SidebarMenuButton>
            </SidebarMenuItem>
          ))}
        </SidebarMenu>
      </SidebarGroupContent>
    </SidebarGroup>
  )
}
```

兩個值得注意的細節：

- **`isActive={currentPath === item.path}`**：用 [TanStack Router](../../../routes/docs/目錄說明.md) 的 `useRouterState()` 拿到目前網址，跟每個選單項目的 `path` 比對——這就是為什麼目前在哪一頁，側邊欄對應的選單項目會有「選中」的樣式。
- **手機版自動收起選單**：`isMobile` 來自 [`useSidebar()`](../../ui/docs/目錄說明.md)（內部用 [`useIsMobile()`](../../../hooks/docs/目錄說明.md)）。手機上側邊欄是滑出式抽屜，點選單項目導頁後，順手呼叫 `setOpenMobile(false)` 把抽屜收起來，否則畫面會卡在「抽屜開著＋已經換頁」的狀態。

### `User.tsx` — 左下角的使用者選單

顯示目前登入者的頭像（`Avatar` + [`getInitials()`](../../docs/目錄說明.md) 算出來的姓名縮寫文字，例如 "Ryan Yang" → "RY"）、姓名、email；點擊展開 `DropdownMenu`，提供兩個動作：

- 「User Settings」→ `<RouterLink to="/settings">`，導向 [settings.tsx](../../../routes/_layout/docs/目錄說明.md)。
- 「Log Out」→ 呼叫 [`useAuth().logout()`](../../../hooks/docs/目錄說明.md)（清除 token、導回登入頁）。

跟 `Main.tsx` 一樣，點擊選單項目後若在手機版會 `setOpenMobile(false)` 收起抽屜。`if (!user) return null`：使用者資料還沒載入完成前，這個區塊不顯示任何東西。
