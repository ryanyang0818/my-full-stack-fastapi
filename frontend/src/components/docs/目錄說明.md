# src/components/ 說明

所有 React 元件的根目錄，依用途分成 UI primitives（`ui/`）、共用元件（`Common/`）與各功能模組（`Header/`、`Admin/`、`Items/`、`Pending/`、`Sidebar/`、`UserSettings/`）。

## 總覽

| 項目 | 類型 | 分類 | 說明 |
|---|---|---|---|
| ui/ | 目錄 | 套件架構本身 | shadcn/ui 風格的 UI primitives（Button、Dialog、Table...） |
| Common/ | 目錄 | 套件架構本身 | 跨頁面共用元件（Logo、Footer、AuthLayout、錯誤/404 頁、DataTable） |
| Header/ | 目錄 | 專案擴充 | 登入後共用的雙層全寬頂欄（AppMenuBar、UserHeader） |
| Sidebar/ | 目錄 | 套件架構本身 | 左側導覽列（AppSidebar）與使用者選單 |
| Admin/ | 目錄 | 套件架構本身 | `/admin` 頁面用：使用者新增/編輯/刪除、表格欄位定義 |
| Items/ | 目錄 | 套件架構本身 | `/items` 頁面用：項目新增/編輯/刪除、表格欄位定義 |
| Pending/ | 目錄 | 套件架構本身 | 資料載入中顯示的 skeleton 元件 |
| UserSettings/ | 目錄 | 套件架構本身 | `/settings` 頁面的三個分頁元件 |
| theme-provider.tsx | 檔案 | 套件架構本身 | 全站亮 / 暗主題管理（Context + localStorage） |

各子目錄的詳細內容，請見其各自的 `docs/目錄說明.md`。

## 重點檔案：`theme-provider.tsx` — 亮 / 暗主題是怎麼運作的

[main.tsx](../../docs/目錄說明.md) 在最外層包了一層：

```tsx
<ThemeProvider defaultTheme="dark" storageKey="vite-ui-theme">
```

這個檔案就是這個 `ThemeProvider` 的實作，提供三樣東西給整個 App：`theme`（使用者選的：`"light" | "dark" | "system"`）、`resolvedTheme`（最終算出來是亮還是暗）、`setTheme`（切換主題的函式）。任何元件都可以用 `useTheme()` 拿到這三者，例如 [Appearance.tsx](../Common/docs/目錄說明.md) 的主題切換開關。

#### 1. 啟動時：從 `localStorage` 還原使用者上次的選擇

```ts
const [theme, setTheme] = useState<Theme>(
  () => (localStorage.getItem(storageKey) as Theme) || defaultTheme,
)
```

如果使用者之前選過「淺色」，重新整理頁面後仍然記得；沒選過的話，就用 `main.tsx` 傳入的 `defaultTheme="dark"` 當預設值。

#### 2. `"system"` 是什麼意思：跟隨作業系統設定

```ts
const getResolvedTheme = (theme: Theme): "dark" | "light" => {
  if (theme === "system") {
    return window.matchMedia("(prefers-color-scheme: dark)").matches ? "dark" : "light"
  }
  return theme
}
```

如果使用者選的是「system」（跟隨系統），就去問瀏覽器「你的作業系統現在是不是深色模式？」（`prefers-color-scheme: dark`）。這就是為什麼有些網站你的 Windows 切換深色模式後，網站也會跟著變——原理就是這個 media query。

#### 3. 真正套用主題：操作 `<html>` 的 class

```ts
const updateTheme = (newTheme: Theme) => {
  const root = window.document.documentElement
  root.classList.remove("light", "dark")

  if (newTheme === "system") {
    const systemTheme = window.matchMedia("(prefers-color-scheme: dark)").matches ? "dark" : "light"
    root.classList.add(systemTheme)
    return
  }
  root.classList.add(newTheme)
}
```

回頭看 [index.css](../../docs/目錄說明.md)：`:root { ... }` 定義亮色變數、`.dark { ... }` 定義暗色變數。這個函式做的事就是**幫 `<html>` 標籤加上或拿掉 `dark` 這個 class**——一旦 `<html class="dark">`，所有 `--background`、`--foreground` 等 CSS 變數的值就會切換成 `.dark` 區塊裡定義的數值，全站顏色瞬間切換。

#### 4. 即時跟隨系統切換

```ts
useEffect(() => {
  updateTheme(theme)
  setResolvedTheme(getResolvedTheme(theme))

  const mediaQuery = window.matchMedia("(prefers-color-scheme: dark)")
  const handleChange = () => {
    if (theme === "system") {
      updateTheme("system")
      setResolvedTheme(getResolvedTheme("system"))
    }
  }
  mediaQuery.addEventListener("change", handleChange)
  return () => mediaQuery.removeEventListener("change", handleChange)
}, [theme, updateTheme, getResolvedTheme])
```

如果使用者選的是「system」，這裡會註冊一個監聽器：**使用者在用網站的過程中，直接去改 Windows 的深色模式設定**，網站會立刻跟著變，不需要重新整理頁面。

#### 5. 切換主題：同時更新畫面與記憶

```ts
setTheme: (theme: Theme) => {
  localStorage.setItem(storageKey, theme)
  setTheme(theme)
}
```

使用者點擊切換主題的開關時，呼叫的就是這個函式：**先存進 `localStorage`（下次開啟記得），再更新 React state**（觸發上面的 `useEffect` 重新套用 class）。

**一句話總結**：`theme-provider.tsx` 把「使用者選了什麼」「系統現在是什麼」「`<html>` 該不該加 `dark` class」三件事的同步邏輯封裝起來，其餘元件只需要呼叫 `useTheme()` 讀狀態、`setTheme(...)` 改狀態。
