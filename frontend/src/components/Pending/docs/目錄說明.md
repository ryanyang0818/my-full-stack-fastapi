# src/components/Pending/ 說明

[items.tsx](../../../routes/_layout/docs/目錄說明.md)、[admin.tsx](../../../routes/_layout/docs/目錄說明.md) 用 `useSuspenseQuery` + `<Suspense>` 載入資料時（見 [routes/_layout/docs/目錄說明.md](../../../routes/_layout/docs/目錄說明.md) 的 `items.tsx` 解析），資料還沒回來之前顯示的「骨架表格」。

## 總覽

| 項目 | 類型 | 分類 | 說明 |
|---|---|---|---|
| PendingItems.tsx | 檔案 | 套件架構本身 | `/items` 載入中的骨架表格（4 欄） |
| PendingUsers.tsx | 檔案 | 套件架構本身 | `/admin` 載入中的骨架表格（5 欄） |

---

## 兩者共同模式：用 `<Skeleton>` 拼出「假表格」

```tsx
const PendingItems = () => (
  <Table>
    <TableHeader>
      <TableRow>
        <TableHead>ID</TableHead>
        <TableHead>Title</TableHead>
        <TableHead>Description</TableHead>
        <TableHead><span className="sr-only">Actions</span></TableHead>
      </TableRow>
    </TableHeader>
    <TableBody>
      {Array.from({ length: 5 }).map((_, index) => (
        <TableRow key={index}>
          <TableCell><Skeleton className="h-4 w-64 font-mono" /></TableCell>
          <TableCell><Skeleton className="h-4 w-32" /></TableCell>
          <TableCell><Skeleton className="h-4 w-48" /></TableCell>
          <TableCell><div className="flex justify-end"><Skeleton className="size-8 rounded-md" /></div></TableCell>
        </TableRow>
      ))}
    </TableBody>
  </Table>
)
```

關鍵點：

- **表頭是真的、表身是假的**：`<TableHeader>` 用真正的欄位標題（ID/Title/Description...），讓使用者一開始就能看出這個表格將會顯示什麼；`<TableBody>` 則用 `Array.from({ length: 5 })` 產生 5 列灰色的 [`<Skeleton>`](../../ui/docs/目錄說明.md) 色塊（[skeleton.tsx](../../ui/docs/目錄說明.md) 就是一個會閃爍動畫的灰色 `<div>`）。
- **`Skeleton` 的寬度模擬真實內容的長寬比**：例如 ID 欄用 `w-64 font-mono`（模擬一串 UUID 的寬度），Title 欄用較窄的 `w-32`，最後一欄是 `size-8 rounded-md` 模擬操作按鈕的方塊——讓載入完成的瞬間，畫面從「骨架」換成「真實資料」時跳動感最小。
- **與 `<Suspense>` 的搭配**：

  ```tsx
  <Suspense fallback={<PendingItems />}>
    <ItemsTableContent />
  </Suspense>
  ```

  `ItemsTableContent` 內部用 `useSuspenseQuery` 拿資料，資料沒回來時 React 會直接渲染 `fallback`——也就是這裡的 `<PendingItems />`。詳細機制請見 [routes/_layout/docs/目錄說明.md](../../../routes/_layout/docs/目錄說明.md)。

`PendingUsers.tsx` 結構完全相同，只是多一欄（Full Name / Email / Role / Status / Actions，對應 [Admin/columns.tsx](../../Admin/docs/目錄說明.md) 的 5 欄），且 Role/Status 欄用了圓角 `Skeleton` 模擬 [`<Badge>`](../../ui/docs/目錄說明.md) 與狀態圓點的形狀。
