# src/hooks/ 說明

放置可在多個元件間共用的 React hooks：登入認證、全域提示、複製到剪貼簿、行動裝置偵測。

## 總覽

| 項目 | 類型 | 分類 | 說明 |
|---|---|---|---|
| useAuth.ts | 檔案 | 套件架構本身 | 登入 / 註冊 / 登出，以及目前登入使用者資料 |
| useCopyToClipboard.ts | 檔案 | 套件架構本身 | 把文字複製到系統剪貼簿 |
| useCustomToast.ts | 檔案 | 套件架構本身 | 統一風格的成功 / 失敗提示訊息 |
| useMobile.ts | 檔案 | 套件架構本身 | 判斷目前畫面是不是行動裝置寬度 |

---

## 重點檔案：`useAuth.ts` — 整個 App 的「登入狀態」中心

這個 hook 把「登入、註冊、登出、目前是誰登入」全部包在一起，任何元件只要 `const { user, logout } = useAuth()` 就能用。

#### `isLoggedIn()` — 最簡單的判斷：有沒有 token？

```ts
const isLoggedIn = () => {
  return localStorage.getItem("access_token") !== null
}
```

這個函式**沒有放在 `useAuth` 裡面**，而是獨立 `export`。原因是：路由守衛（例如 [routes/_layout.tsx](../../routes/) 或 [routes/login.tsx](../../routes/)）需要在「進入頁面之前」就判斷要不要導向登入頁，這個時機在 React component 渲染之前，所以不能用 hook，只能用一個普通函式。

#### `user` — 目前登入的使用者資料

```ts
const { data: user } = useQuery<UserPublic | null, Error>({
  queryKey: ["currentUser"],
  queryFn: UsersService.readUserMe,
  enabled: isLoggedIn(),
})
```

這裡的關鍵是 `enabled: isLoggedIn()`：**如果連 token 都沒有，就連 API 都不打**，避免一進首頁就先打一次注定 401 的請求。`UsersService.readUserMe` 是 [client/](../../client/) 自動產生的 API 呼叫（對應後端 `GET /users/me`）。任何元件想知道「目前登入的是誰」（例如顯示 email、判斷是不是 superuser），都從這裡拿。

#### `signUpMutation` — 註冊

```ts
const signUpMutation = useMutation({
  mutationFn: (data: UserRegister) =>
    UsersService.registerUser({ requestBody: data }),
  onSuccess: () => {
    navigate({ to: "/login" })
  },
  onError: handleError.bind(showErrorToast),
  onSettled: () => {
    queryClient.invalidateQueries({ queryKey: ["users"] })
  },
})
```

註冊成功 → 導去登入頁；失敗 → 用 [utils.ts 的 handleError](../docs/目錄說明.md) 把錯誤訊息丟到 toast。`onSettled` 不管成功失敗都會跑，用來刷新使用者列表（給 admin 頁面用）。

#### `loginMutation` — 登入

```ts
const login = async (data: AccessToken) => {
  const response = await LoginService.loginAccessToken({ formData: data })
  localStorage.setItem("access_token", response.access_token)
}

const loginMutation = useMutation({
  mutationFn: login,
  onSuccess: () => {
    navigate({ to: "/" })
  },
  onError: handleError.bind(showErrorToast),
})
```

故事流程：使用者在 [routes/login.tsx](../../routes/) 填完帳密 → 呼叫 `loginMutation.mutate({...})` → 內部打 `LoginService.loginAccessToken`（對應後端 OAuth2 password flow）→ 後端回傳 `access_token` → **存進 `localStorage`** → 導回首頁 `/`。從此之後，[main.tsx](../docs/目錄說明.md) 設定的 `OpenAPI.TOKEN` 就會從這個 `localStorage` 讀到 token，後續所有 API 請求都自帶這個 token。

#### `logout()` — 登出

```ts
const logout = () => {
  localStorage.removeItem("access_token")
  navigate({ to: "/login" })
}
```

跟 [main.tsx](../docs/目錄說明.md) 裡「API 回 401/403 自動登出」的邏輯做的事情一樣（清掉 token、導回登入頁），只是這裡是**使用者主動點擊「登出」**觸發的。

**一句話總結**：`useAuth` 是「登入相關所有操作」的唯一入口；`isLoggedIn()` 是「路由守衛用的快速通行證檢查」。

## 其他檔案說明

### `useCustomToast.ts` — 全站統一的成功 / 失敗提示

```ts
const useCustomToast = () => {
  const showSuccessToast = (description: string) =>
    toast.success("Success!", { description })

  const showErrorToast = (description: string) =>
    toast.error("Something went wrong!", { description })

  return { showSuccessToast, showErrorToast }
}
```

包裝了 [sonner](https://sonner.emilkowal.ski/) 這個 toast 套件，固定好標題文字（成功一律是「Success!」、失敗一律是「Something went wrong!」），各元件只要負責傳「描述」這一句話。`showErrorToast` 最常見的用法是搭配 [utils.ts 的 handleError](../docs/目錄說明.md)：`onError: handleError.bind(showErrorToast)`。

### `useCopyToClipboard.ts` — 一鍵複製文字

```ts
const [copiedText, copy] = useCopyToClipboard()

<button onClick={() => copy("hello@example.com")}>複製 Email</button>
```

`copy(text)` 會把文字寫進系統剪貼簿，成功的話 `copiedText` 會變成那段文字（可以用來顯示「已複製！」的提示），**2 秒後自動恢復成 `null`**。如果瀏覽器不支援 `navigator.clipboard`（例如非 HTTPS 環境），會印警告並回傳 `false`，不會讓整個 App 壞掉。註解標明來源是 [usehooks-ts.com](https://usehooks-ts.com/react-hook/use-copy-to-clipboard) 的公開範例。

### `useMobile.ts` — 現在是不是「手機畫面」？

```ts
const MOBILE_BREAKPOINT = 768

export function useIsMobile() {
  // ...
  const mql = window.matchMedia(`(max-width: ${MOBILE_BREAKPOINT - 1}px)`)
  // ...
  return !!isMobile
}
```

用瀏覽器的 `matchMedia` API 監聽「視窗寬度是否小於 768px」，回傳一個會即時更新的布林值。例如 [components/ui/sidebar.tsx](../../components/ui/) 用它來判斷：寬螢幕時側邊欄是「固定展開的欄位」，窄螢幕（手機）時側邊欄變成「點擊才彈出的抽屜（Sheet）」。
