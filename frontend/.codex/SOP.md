# Codex SOP

## 計畫

- 當使用者要求「提出計畫」時，輸出 `<proposed_plan>` 區塊，讓 Codex app 嘗試在右側顯示可修改計畫。
- 計畫需要包含目標、主要變更、驗收方式與假設。
- 計畫的測試項目不要包含 `npm run build`。

## 計畫保存

- 當使用者允許計畫後，把計畫存入 `docs/執行計畫/`。
- 檔名建議使用 `YYYY-MM-DD-短標題.md`。
- 若資料夾不存在，先建立資料夾。

## 歸檔

- 當使用者要求「執行歸檔」時，把對應計畫移入 `docs/執行計畫/完成計畫/`。
- 不刪除已完成計畫。

## 溝通

- 回覆使用繁體中文。
- 保持短、專業、具體。
- Markdown 表格用於整理關係、流程與比較。
- 重要開發任務先說明計畫，再進行實作。
